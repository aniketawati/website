EasyTags: Stop Hand-Writing Go Struct Tags

A tiny CLI that generates your json/xml/sql tags so you don't have to

21 February 2015
Project on github.com/betacraft/easytags.

A few weeks after YAAG, another small Go annoyance started to grate. You write struct fields in CamelCase, because that's how Go wants them. But when those structs marshal to JSON (or XML, or hit a SQL column) you almost always want snake_case. So you end up typing every field name twice — once as the Go field, once as the tag, in a different case — for every field, in every struct, forever.

It's exactly the kind of repetitive, mechanical work that a computer should be doing. So we made the computer do it. EasyTags is a tiny CLI that walks every struct declaration in your file and adds the tags for you, converting the field name to snake_case (camel and pascal cases are supported too).

Usage

Point it at a file and tell it which tags you want:

easytags {file_name} {tag_name_1:case_1, tag_name_2:case_2}

# the simplest form — defaults to json, snake_case:
easytags file.go

Even better: with go generate

Drop a directive at the top of your source file:

//go:generate easytags $GOFILE json,xml,sql

Then run go generate, and every struct in the file gets its json, xml, and sql tags written for you. It's also polite about your intent: if you've already set a tag's value to "-", EasyTags leaves it exactly as it is.

Small tool, big quality of life

EasyTags will never make a highlight reel — it's about 75 stars' worth of "oh, that's nice." But removing a small, constant friction from everyday coding is its own kind of win. The best tools are often the ones you forget you're using.
