YAAG: API Docs for Go Without Writing a Single Comment

Yet Another API doc Generator — a weekend hack from GopherGala 2015

23 January 2015
Project on github.com/betacraft/yaag.

Go is a wonderful language for building web apps. By early 2015 the ecosystem had everything you could want — fast HTTP servers, a dozen good routers, helper libraries for days. Everything, that is, except a decent way to document your APIs. And let's be honest: nobody has ever enjoyed writing API documentation.

So at GopherGala 2015 — a 48-hour global Go hackathon — we decided to fix the part everyone quietly hated. We built the first API doc generator for Go web apps and, true to the genre of "yet another <thing>," called it YAAG: Yet Another API doc Generator.

The premise: zero comments

Most doc tools ask you to annotate every handler with special comments, keep them in sync, and pray. We wanted the opposite. The whole pitch of YAAG was that you shouldn't have to write a single line of comments to get usable docs.

How it works

YAAG is middleware. You drop its handler into your routes and then just use your API the way you already do — hit it with Postman, curl, your test suite, the frontend, whatever. YAAG watches the real requests and responses go by and continuously writes out a clean, Bootstrap-based apidoc.html (plus a JSON file capturing every call it saw). Your documentation is literally a side effect of exercising your own API.

With the standard net/http package it's about four lines:

func handler(w http.ResponseWriter, r *http.Request) {
    fmt.Fprintf(w, "Hi there, I love %s!", r.URL.Path[1:])
}

func main() {
    yaag.Init(&yaag.Config{On: true, DocTitle: "Core", DocPath: "apidoc.html"})
    http.HandleFunc("/", middleware.HandleFunc(handler))
    http.ListenAndServe(":8080", nil)
}

From there it grew adapters for the popular frameworks of the day — Gorilla Mux, Martini, and Revel — so most Go web apps could bolt it on with a couple of lines.

The afterlife

We built it for in-house docs — the kind you need for your own team but don't want to hand-craft for the public. People liked it more than we expected. Last I checked it's sitting at around 286 stars and a healthy pile of forks, which for a weekend hackathon project is genuinely humbling. Not bad for something whose name is a shrug.
